# DFT-First Orchestration

The project graph is driven by the observable, not by software menus. A typical graph is:

```text
question -> structure campaign -> method/convergence design -> engine tasks
-> engine validation -> property/wavefunction analysis -> claim gate
-> optional ML/kinetics -> figure/report gate
```

Independent tasks may run concurrently only when they have separate owner directories and immutable input manifests. A failed, inconclusive, or contradicted branch remains visible.
