# DFT Validation Ladder

1. **Planned** — objective, model, method and success criteria exist.
2. **Prepared** — immutable inputs and structure/method provenance exist.
3. **Preflight passed** — syntax, files, mappings and site prerequisites pass deterministic checks.
4. **Program completed** — a recognized termination record exists.
5. **Technically validated** — SCF/optimization/frequency/force/path/property-specific checks pass.
6. **Scientifically accepted** — model identity, references, method comparability, sensitivity and magnitude support the observable.
7. **Claim accepted** — the result supports a bounded statement with limitations and falsification conditions.

No later rung is inferred from an earlier rung. Scheduler completion, Gaussian `Normal termination`, VASP timing output, QE `JOB DONE`, or CP2K `PROGRAM ENDED AT` only establish program completion.
